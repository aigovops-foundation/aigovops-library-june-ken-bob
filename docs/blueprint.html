00 Principles the system is built on

Every architectural choice traces back to one of these. If a feature can't, it doesn't ship.
"Agents do the bureaucracy; humans hold the meaning — and humans hold the keys."
  • Center of truth, convene anywhere. The Foundation runs one governed core that is the canonical source of policy, evidence, and identity — but it shows up wherever the community already is (web, chat, email, their own agents), never demanding they come to us.
  • Portable by construction — no SaaS lock-in. The exact same container runs on a laptop offline, a $5 VPS, or the canonical cloud. Your data and your evidence are yours; you can walk away with everything, signed and verifiable.
  • Local-first & private by default. Nothing leaves the machine unless you choose the cloud. Beacons are metadata-only — no payloads, ever.
  • Propose, don't execute. Agents draft; humans approve anything irreversible. Capability is a reversible dial that widens with trust, never a blank cheque.
  • The receipt is the unit of truth. Every meaningful action produces an append-only, Ed25519-signed OVERT record. If it isn't signed, it didn't happen.
  • English-first, global-always. The default is English, but every human and every agent speaks the member's own language. Localization is not a feature flag — it's the substrate.
  • Delight is a requirement, not a polish pass. The library is warm, fast, and humane on purpose. A correct answer that feels cold has half-failed.

03 Backend — the governed core

Nine services behind one gateway. Node 20+ / pnpm, Apache-2.0, all swappable.
FIG 03 — the core's services; every one is open-source and replaceable, talking through the gateway, persisting to one ledger

POLICY / YES-GATE ENGINE

node · pure functions · deterministic

The heart. Turns each constraint into a Yes/No gate carrying its lifecycle — Get / Stay / Recover to Yes. Stateless and testable; the same evaluation runs client-side for preview and server-side for the record.

LANTERN COMPILER

framework library → YAML controls → OVERT

Compiles the mapped frameworks (40 mapped, 23 machine-readable and growing — NIST AI RMF, EU AI Act, ISO 42001, SOC 2, GDPR, Colorado AI Act, Korea AI Basic Act, NYC LL144, India DPDP…) into executable controls and signed bundles.

BEACON

Ed25519 + JCS (RFC 8785) + NDJSON · openssl-verifiable

Signs Model / Prompt / Artifact beacons — metadata-only, no payloads — into an append-only ledger. Verification needs nothing but a public key and openssl. Steward: OVERT 1.0, profile aigovops-beacon.v1.

AGENT RUNTIME

MCP host · propose-not-execute · sandboxed tools

Runs the agents that do the bureaucracy. Tools are capability-scoped; anything irreversible returns a proposal for a human gate, never an action. This is where "agents convene, humans decide" lives in code.

MODEL ROUTER

Ollama (local, default) · cloud providers (opt-in)

Cheap/local model by default for snappy, free responses; escalates to a stronger cloud model only when the task needs it and the member opted into cloud. Caches "Yes" results so a repeat answer is instant and $0.

IDENTITY & MEMBERSHIP

OIDC · roles · the capability dial

Library Card = membership. Roles map to a reversible capability dial (read → propose → auto-act-within-caps) that widens with demonstrated trust and can be turned down instantly.

i18n SERVICE

ICU message catalogs · RTL · machine-translation fallback

Negotiates locale at the gateway; serves UI strings, agent system-prompts, and voice in the member's language. Human-reviewed translations are signed like any other artifact.

EVIDENCE STORE

append-only NDJSON · content-addressed · exportable

The one source of truth. Every receipt lands here, immutable; exportable as a verifiable bundle so a member (or auditor) can take the whole history and check it offline.

04 Frontend — the rooms

Surfaces are thin. The intelligence is in the core; the room is the welcome.

Each room is a view onto a core capability, built static-first (GitHub Pages + Cloudflare today) and talking to the core through the gateway via the established PROXY_URL pattern — no API keys ever in client JS. A keyword intent classifier routes the conversation; a voice layer adds speech in the local language; everything renders right-to-left and re-strings itself per locale.

  • Front Desk → Policy + Agent runtime. Conversational concierge; signed answers with a "view receipt" link.
  • Reading Room → Lantern + Policy + the proof set. Bring a hard problem; see risk, the gates you'll face, real fail cases, and the path to Yes.
  • Makerspace (Design Studio) → Agent runtime + Beacon. Drag gates, agents, and guards onto a canvas; live cost + safety preview; test, then deploy as a signed service.
  • The Commons → Evidence store + registry. Pull and remix signed, secrets-stripped workflows; the library compounds.
  • Human Library & Coffee Shop → Identity + Federation. Check out a real mentor; agent-made introductions over global tables. Humans connect; agents convene.

Surfaces are disposable; the core is durable. A room can be redesigned, translated, or re-skinned per branch without touching policy or evidence.

05 Agent-first architecture — human & agent, in your language

Agents are the staff of the library. Members are the citizens. Both speak the local tongue.

The system is agent-operated and human-governed. Agents handle intake, framework mapping, evidence assembly, monitoring, and introductions — the bureaucracy. Humans hold the gates that carry meaning and risk.

  • Propose-not-execute. Agents emit proposals (a diff, a draft, a plan). Irreversible effects pause at a human gate with a one-line plain-language summary in the member's language.
  • Flag-not-delete, cap-and-pause. Destructive operations are quarantined, not performed; spend and blast-radius are capped, and the agent pauses at the cap rather than pushing through.
  • Build-your-own-agent, governed. Members compose agents in the Makerspace; those agents inherit the same gates, caps, and signing — capability is a dial they earn, not a switch they flip.
  • Bilingual by design. The agent's system prompt, its reasoning surface, and its replies are localized; an Arabic or Yoruba member converses end-to-end in Arabic or Yoruba, and the receipt is still canonical and verifiable.
  • MCP both ways. The core is an MCP host (agents use tools) and an MCP server (a member's own agent can consult the Foundation as a governed tool) — convening anywhere, on our principles.

07 Safety

Held to a higher bar — we govern AI, so we start conservative and earn width.
  • The irreversibility boundary. Agents never perform destructive or external-effect actions autonomously; those become proposals behind a human gate. The boundary is enforced in the agent runtime, not left to prompt discipline.
  • Capability dial + caps. Every member and agent has an explicit, reversible capability level and hard caps on spend and blast radius. Defaults are narrow; trust widens them; one toggle narrows them again.
  • Content & community safety. Layered checks at input and output; conservative defaults on anything touching minors, weapons, or harm. Moderation at scale stays human-in-the-loop — agents triage, people decide.
  • Sandboxed tools. Agent tools run with least privilege in isolated contexts; no ambient credentials, no network egress beyond declared endpoints.
  • Kill switch & rollback. Any deployed workflow can be paused instantly; because effects are signed and reversible-by-design, recovery is "Recover to Yes," not a forensic scramble.

08 Security

Secrets never touch the client. Trust is cryptographic, not promissory.
  • Key custody. Ed25519 signing keys live server-side — env/secret-store locally, KMS/HSM in the canonical cloud — and rotate on schedule. Public keys are published so anyone can verify.
  • No keys in client JS. The browser talks only to the gateway/proxy (the proven PROXY_URL + Cloudflare Worker pattern) with a strict CORS allow-list; model and provider credentials stay in the core.
  • AuthN / AuthZ. OIDC for identity; role- and capability-based authorization checked at the gateway and re-checked at each service (defense in depth, least privilege).
  • Supply chain. Dependencies pinned; framework bundles and shared workflows are Beacon-signed, so a pulled Commons artifact is verifiable before it runs. SBOM published.
  • Transport & isolation. TLS everywhere (Caddy/Cloudflare); per-tenant data isolation; rate-limiting and abuse detection at the edge.
  • Threat model is written down. Documented and reviewed: prompt injection, exfiltration via tools, key compromise, malicious shared workflows, federation impersonation — each with a control above.

09 Privacy

The most private option is always the default. Convene without harvesting.
  • Local-first. Run entirely on your machine and nothing leaves it. Cloud is an explicit, per-action opt-in shown as a receipt line item.
  • Metadata-only beacons. Beacon records the fact and shape of an action, never the payload — no prompts, no documents, no PII in the ledger.
  • Secrets stripped on share. Publishing to the Commons runs a redaction pass; PII guard and secret-scanning are mandatory gates, not optional.
  • Data minimization & rights. Collect the least; honor export and delete; align to GDPR, India DPDP, and regional regimes by construction (they're in the framework library, so we hold ourselves to them).
  • Member secrets hygiene. As a community we now hold each other's governance context — treated as sensitive by default, never cross-shared, never used to train anything.
  • Federation privacy. When we convene inside someone else's channel, we take only what the task needs and write nothing back we weren't invited to.

10 Logging & observability

The receipt is the log. The log is the proof. The proof is exportable.
  • The signed evidence ledger is the primary log: append-only NDJSON, Ed25519-signed, content-addressed, openssl-verifiable. Hand an auditor the bundle and they can check it without us.
  • Audit trail. Who proposed, who approved, which gate, which framework, which model, what it cost — every step of Fig 04 leaves a record tied to the receipt.
  • Immutable fact log (HIBT pattern). Key decisions and verifications are captured in a dated, version-controlled fact log so the project's own history is itself auditable.
  • App observability. Structured logs, health checks, and a public /status dashboard; alerting on error rates, gate failures, and cap breaches.
  • Privacy-preserving analytics. Plausible-style, cookieless, aggregate-only — we measure usage without surveilling members.

11 Global & i18n — English-first, never English-only

A member in Lagos, São Paulo, or Bengaluru gets the full experience in their language — including the agent.
  • Localization is the substrate. ICU message catalogs for UI; localized agent prompts and replies; full RTL; locale-aware dates, numbers, and currency. English is the source-of-truth string; everything else derives from it.
  • Translation, governed. Machine translation gives instant coverage; human-reviewed translations are signed artifacts that supersede MT — quality you can verify.
  • Voice in-language. Speech in and out in the member's language at the Front Desk and beyond.
  • Per-branch culture. Each global branch carries its own warmth — palette, music, soundscape, and local payment rails (UPI, Pix, M-Pesa, SEPA, Alipay…) on the Stripe + PayPal + regional backbone.
  • Canonical evidence stays universal. However it's spoken, the receipt is one canonical, machine-checkable record — language never fragments the truth.

12 Delight — engineered, not decorated

The library metaphor is a UX architecture, and speed is part of it.
  • Latency budgets. Local cheap-model default and cached-Yes responses keep the common path instant and free; the cloud is reserved for when it truly earns its cost.
  • The rooms as information architecture. Capabilities map to places people understand — a front desk, a reading room, a workshop, a café — so the system is legible without a manual.
  • Warmth & light. The visual language (warm light, the Beacon Eye, the journey-to-Yes fresco) and the per-branch soundscape make governance feel like belonging, not bureaucracy.
  • Humane copy, in your language. Plain-language gate summaries, honest cost receipts, and "here's your path to Yes" framing — never a wall of red.
  • Micro-interactions. Signed-receipt confirmations, streaks in the practitioner quiz, the agent's gentle "shall I?" — small moments that say you're cared for here.
Delight and rigor aren't in tension — a system you trust and enjoy is the one people actually use to get to Yes.

13 Shipping the connections

From this blueprint to a running instance — the path, with the human in the right seats.
  • Package the core as one container with compose for the datastore + ledger; one-command local run on OSS models.
  • Stand up the canonical instance on the chosen host with KMS-held keys, multi-region models, the public /status and published verification keys.
  • Wire the rooms to the gateway via PROXY_URL; ship i18n catalogs (EN first), add locales as branches come online.
  • Add federation adapters one channel at a time — web+voice, then Telegram/Discord/email/Teams, then the MCP server so members' own agents can convene us.
  • CI/CD & tests. GitHub Actions gate every change through the Cloud-Mary suite before deploy; signed releases.
  • Irreversibility boundary holds for us too. I prepare configs, values, and verification; you and Ken make the irreversible clicks — DNS, registrar, key enrollment, account creation — after I flag the risk.
