How they connect

The Demo says why. The Design Book shows the feel. The Blueprint is what to build. The Plan is how we operate — the agents, skills, and processes that produce all of it. And the Core is the running skeleton those processes ship to. Every piece bends toward the same spine, and every meaningful action leaves a signed receipt.

Run it yourself

The core is one dependency-free Node service — $0, local-first, no SaaS. Run it three ways; each boots with its own Ed25519 key and a signed, append-only ledger. This page is the static story — the commands below run the live core, with an interactive Control Room at /console. Open the live Control Room →

Local · Node ≥18

On your machine

$ cd core
$ npm start
→ http://localhost:8787/console
Docker

As a container

$ cd core
$ docker compose up --build
one image; keys + ledger persist on a volume →
Fly.io · hosted

Any moment, anywhere

$ fly apps create <name> --org personal
$ fly volumes create aigov_data -r iad -s 1
$ fly deploy
● live → aigovops-library-core.fly.dev/console
Verify it, offline. npm run export:evidence writes a bundle anyone can check with openssl or node — no network, no trust in our code. Full steps live in the repo's DEPLOY.md. The governed core is Apache-2.0; agents propose, humans hold the keys.
