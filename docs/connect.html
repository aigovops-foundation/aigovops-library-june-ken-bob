AIGOVOPS FOUNDATION · MCP CONNECTOR

Govern Any Agent

Point any MCP client at the governed core, and your agent's irreversible actions route through the Yes-Gate — each one paused for a human and sealed with a signed receipt.

→ The loop your agent inherits

No new trust to configure — the agent simply can't act irreversibly without a human's yes.
propose → human decide → brokered scoped token → sandboxed run → verify

Your coding agent (Claude Code, Claude Desktop, Cursor — anything that speaks MCP) keeps doing what it does. But when it reaches for something irreversible — a deploy, a secret, a destructive write — that action becomes a proposal a human approves. On approval the gate mints a short-lived, scoped credential; the agent never sees the master secret. The tool runs sandboxed. Every step leaves a metadata-only, Ed25519-signed receipt. A steward holds a global kill switch.

→ Install — zero dependencies

The core is dependency-free; you need only Node ≥ 20.

Claude Code:

claude mcp add aigovops-governed-core -- node /ABS/PATH/core/scripts/mcp-server.mjs

Claude Desktop / Cursor / any MCP client: copy core/connector/mcp.json into your client's config (replace <ABS_PATH>), or run it directly:

cd core && npm run mcp # speaks MCP (JSON-RPC 2.0) on stdio

The stdio server is a single trusted principal resolved from its launch env — a member by default, a steward with AIGOV_MCP_ROLE=steward. For multi-tenant networked use, point at the HTTP governed API instead, where each caller carries a real OIDC identity.

→ What stays true, no matter the agent

  • An agent never receives a raw secret — only a short-lived, scoped token.
  • Anything irreversible pauses for a human; a denial brokers nothing.
  • Every action is a signed, metadata-only receipt — no prompts, no payloads, no PII.
  • A steward can arm the global kill switch and halt the loop, fail-closed.
Agents do the bureaucracy; humans hold the meaning — and humans hold the keys.
