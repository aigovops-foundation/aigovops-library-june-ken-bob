# UX + accessibility pass — Library site

*2026-06-28. Staged, not pushed. Verified in the local preview (no visual regression).*

## What the governed audit found

The core's `accessibility-audit` skill (`core/src/core/a11y.js`, WCAG2.2-AA *subset*) scored
**21/21 pages at 100** — clean on the automatable checks (alt text, lang, title, viewport,
single h1, no empty links). That's the floor, and it was already met.

## What the subset can't see — and what I fixed

The subset explicitly excludes motion, focus order, landmarks, and contrast. Scanning for
those showed they were present on only `rooms.html`. Applied:

- **Reduced motion** (`prefers-reduced-motion: reduce`) — **21/21 pages**. Respects the OS
  "reduce motion" setting; disables the grid/hover animations for vestibular safety
  (WCAG 2.3.3). Purely additive CSS, idempotent (`<style id="a11y-universal">`).
- **Keyboard focus-visible** — **21/21 pages**. A visible 2px outline on links/buttons/inputs
  for keyboard users (WCAG 2.4.7). Was effectively invisible before.
- **Skip-to-content link + `<main>` landmark** — applied to the **hub** as the model
  pattern. Off-screen until focused; jumps past the Jeeves hero to the content.

## Follow-ups (staged none — flagged for a decision)

1. **Roll skip-link + `<main>` to the other 20 pages.** Their layouts vary (deep-docs have a
   sticky nav; demos differ), so a blanket insert is riskier than the universal CSS — better
   done per-template. ~20 small structural edits.
2. **Contrast spot-check.** The navy/teal/gold palette is likely AA for body text, but
   `--ink2` (#9fc0bd) muted text on the gradient and gold-on-navy should be measured with a
   real contrast tool (the static subset can't). Candidate for the axe/pa11y opt-in profile
   the code already contemplates.
3. **Reduced-motion for the Jeeves widget** (third-party `jeeves-widget.js` / `jeeves-hero.js`)
   — out of this repo's control; worth confirming on the foundation side.
